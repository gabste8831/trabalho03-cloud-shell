Arquitetura de Sistemas

Como está desenhada a infraestrutura FreelaCloud?

Entenda o fluxo completo da aplicação conteinerizada original e a camada operacional Linux criada no Trabalho 03.

🛠️ Arquitetura do Trabalho 02 (Aplicação Original)

O FreelaCloud foi construído utilizando uma arquitetura de microserviços conteinerizada, composta por:

  • Container Frontend/Backend (app): Desenvolvido com Node.js e Next.js (App Router), servindo a interface do usuário e as APIs internas na porta 3000.
  • Container Banco de Dados (db): Servidor relacional PostgreSQL 16 que armazena as vagas, perfis e vínculos contratuais.
  • ORM Prisma: Abstração de banco que gera consultas tipadas e manipula migrações de esquema automáticas.
  • pgdata (Volume Persistente): Garante que os cadastros efetuados pelos usuários persistam localmente mesmo se os containers forem recriados.

🐳 Arquitetura do Trabalho 03 (Camada DevOps & Linux)

Neste Trabalho 03, simulamos a administração e o deploy das páginas institucionais de monitoramento do FreelaCloud.

Configuramos uma instância Linux baseada em Ubuntu Server no Docker, que roda o servidor de páginas web Apache (apache2). O deploy é efetuado de forma limpa através de scripts Bash que copiam a build estática para a pasta root do Apache (/var/www/html).

Além disso, para segurança, os scripts criam o usuário do sistema freela_admin e o grupo freela_ops, e utilizam regras de controle de acesso restrito (permissões 770) nas pastas sob /app/freelacloud/ para evitar vazamento de dados de vagas e perfis confidenciais.

📊 Monitoramento e Segurança da Operação

O monitoramento contínuo coleta o consumo de CPU, memória RAM ocupada e espaço em disco livre. Se algum recurso exceder a marca de 80% de consumo ou se o servidor Apache cair, alertas automáticos são exibidos em tela e arquivados nos arquivos de log persistentes.

Toda a saída operacional do ecossistema é consolidada no script de relatório e exportada diretamente para /app/logs/relatorio_execucao.txt, servindo de auditoria para os engenheiros DevOps.
